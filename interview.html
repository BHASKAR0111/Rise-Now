🎤 Interview Prep

Interview Q&A Bank

Most asked interview questions with model answers — practice until you're confident.

HR & Behavioral Questions
Tell me about yourself.
Easy ▾

Keep it professional and structured — present, past, future formula works best.

Model Answer: "I'm a Data Analyst with an MCA from Graphic Era University. I specialize in Python, SQL, and Power BI. I've built projects like a customer churn analysis dashboard and a real-time vehicle detection system. I'm passionate about turning data into decisions that drive real impact, and I'm now looking for a full-time role where I can contribute and grow."

💡 Tip: Keep it under 90 seconds. End with why you want THIS role at THIS company.
What is your greatest weakness?
Medium ▾

Be honest but show self-awareness and growth. Never say "I'm a perfectionist" — it's overused.

Model Answer: "I sometimes spend too much time trying to get every detail perfect before presenting results. I've been working on this by setting time limits and sharing work-in-progress earlier to get feedback. It's helped me move faster without sacrificing quality."

💡 Tip: Pick a real weakness, then show what you're doing to improve it.
Where do you see yourself in 5 years?
Easy ▾

Show ambition but keep it realistic and relevant to the role.

Model Answer: "In 5 years, I see myself as a senior data analyst or analytics lead — someone who not only delivers insights but helps shape data strategy. I want to grow technically with ML and also develop skills in stakeholder management and leadership."

💡 Tip: Always connect your future goals to what the company does.
Why should we hire you?
Medium ▾

This is your sales pitch. Be confident, specific, and match it to the job description.

Model Answer: "I bring a combination of strong technical skills — Python, SQL, Power BI — and real project experience. I've already built end-to-end dashboards analyzing 7,000+ records and can hit the ground running. I'm also a fast learner who adapts quickly, and I'm genuinely passionate about data-driven decision making."

💡 Tip: Use specific numbers and project examples — it makes you memorable.
Tell me about a time you failed and what you learned.
Hard ▾

Use the STAR method: Situation, Task, Action, Result. Interviewers want to see maturity.

Model Answer: "During a project, I underestimated the time needed for data cleaning and missed our initial deadline. I learned to always do a quick data audit before committing to timelines. Since then, I buffer 20-30% extra time for data preparation, which has made my estimates much more accurate."

💡 Tip: End with what you learned and how you changed. Never end on the failure itself.
Data Analytics Questions
What is the difference between a KPI and a metric?
Easy ▾

A metric is any measurable value (page views, clicks, revenue). A KPI (Key Performance Indicator) is a metric that is directly tied to a business goal or objective.

Example: "Revenue" is a metric. "Monthly Revenue Growth of 10%" is a KPI.

💡 Tip: All KPIs are metrics but not all metrics are KPIs.
What steps do you follow when starting a new data analysis project?
Medium ▾

The standard approach:

1. Define the problem — what question are we answering?
2. Collect data — identify sources and gather raw data
3. Clean data — handle nulls, duplicates, outliers
4. Explore (EDA) — distributions, correlations, patterns
5. Analyze — apply statistics or models
6. Visualize & Present — dashboards, charts, storytelling

💡 Tip: Mention a real project where you followed this process.
What is the difference between correlation and causation?
Medium ▾

Correlation means two variables move together — when one goes up, the other tends to go up (or down).

Causation means one variable directly causes the change in another.

Example: Ice cream sales and drowning rates are correlated (both increase in summer) — but ice cream doesn't cause drowning. The real cause is hot weather.

💡 Tip: "Correlation does not imply causation" — this is a classic analyst interview point.
How do you handle missing data?
Medium ▾

Common strategies depending on context:

• Delete rows — if missing data is less than 5% and random
• Mean/Median imputation — fill with average (for numerical)
• Mode imputation — fill with most common value (for categorical)
• Forward/Backward fill — for time series data
• Flag as "Unknown" — keep it as a separate category

💡 Tip: Always explain WHY you chose a specific method — context matters.
Explain what an outlier is and how you would handle it.
Hard ▾

An outlier is a data point that is significantly different from other observations. It can be detected using Z-score, IQR method, or box plots.

How to handle: Remove if it's a data entry error. Keep if it's a genuine extreme value. Cap/floor using winsorization. Use robust models that handle outliers well (e.g. median instead of mean).

💡 Tip: Say "it depends on the context and cause of the outlier" — shows analytical maturity.
SQL Interview Questions
What is the difference between WHERE and HAVING?
Easy ▾

WHERE filters rows BEFORE grouping. HAVING filters groups AFTER the GROUP BY clause.

Example: WHERE salary > 50000 — filters individual rows.
HAVING COUNT(*) > 5 — filters groups with more than 5 records.

💡 Tip: WHERE cannot be used with aggregate functions — HAVING can.
What are the different types of JOINs in SQL?
Medium ▾

• INNER JOIN — returns only matching rows from both tables
• LEFT JOIN — all rows from left table + matching from right
• RIGHT JOIN — all rows from right table + matching from left
• FULL OUTER JOIN — all rows from both tables
• CROSS JOIN — cartesian product of both tables
• SELF JOIN — joining a table with itself

💡 Tip: INNER JOIN is most commonly used in interviews. Know LEFT JOIN well too.
What is the difference between DELETE, TRUNCATE, and DROP?
Medium ▾

• DELETE — removes specific rows, can be rolled back, slow for large tables
• TRUNCATE — removes all rows, faster, cannot be rolled back easily
• DROP — removes the entire table including structure

💡 Tip: DELETE is DML, TRUNCATE is DDL, DROP is DDL — interviewers love this distinction.
What are Window Functions? Give an example.
Hard ▾

Window functions perform calculations across a set of rows related to the current row, without collapsing the result like GROUP BY.

Common window functions: ROW_NUMBER(), RANK(), DENSE_RANK(), LAG(), LEAD(), SUM() OVER()

Example: SELECT name, salary, RANK() OVER (ORDER BY salary DESC) as rank FROM employees

💡 Tip: Window functions are a senior-level topic — knowing this sets you apart.
Python Interview Questions
What is the difference between a list and a tuple in Python?
Easy ▾

• List — mutable (can be changed), uses square brackets [], slower
• Tuple — immutable (cannot be changed), uses parentheses (), faster

Use tuples when data should not change (e.g. coordinates). Use lists when data will change.

💡 Tip: Tuples are hashable and can be used as dictionary keys — lists cannot.
What are Python decorators?
Hard ▾

A decorator is a function that takes another function as input and extends or modifies its behavior without changing its source code.

Example: @login_required in web frameworks — checks if user is logged in before running a function.

💡 Tip: Decorators follow the @wrapper syntax. Common ones: @staticmethod, @classmethod, @property.
How do you handle missing values in Pandas?
Medium ▾

Common Pandas methods for missing values:

• df.isnull().sum() — check missing values
• df.dropna() — drop rows with missing values
• df.fillna(value) — fill with a specific value
• df.fillna(df.mean()) — fill with mean

💡 Tip: Always explain why you chose a specific strategy — shows analytical thinking.
Web Development Questions
What is the difference between CSS Flexbox and Grid?
Medium ▾

• Flexbox — one-dimensional layout (row OR column). Best for aligning items in a single direction.

• Grid — two-dimensional layout (rows AND columns). Best for complex page layouts.

Use Flexbox for navigation bars, buttons, small components. Use Grid for full page layouts.

💡 Tip: They work great together — Grid for layout, Flexbox for components inside.
What is the difference between == and === in JavaScript?
Easy ▾

• == (loose equality) — compares values after type conversion. 5 == "5" is true

• === (strict equality) — compares both value AND type. 5 === "5" is false

💡 Tip: Always use === in real code — it avoids unexpected type coercion bugs.
What is the difference between var, let, and const in JavaScript?
Medium ▾

• var — function-scoped, can be re-declared, hoisted (old way, avoid)
• let — block-scoped, can be reassigned, not re-declared
• const — block-scoped, cannot be reassigned after declaration

💡 Tip: Use const by default, use let when you need to reassign. Never use var in modern JS.
